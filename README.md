Planejamento de Interface: Wireframes Açaí Itaki

1. Metodologia de Desenho Manual
A fase de wireframing foi conduzida através de esboços manuais para permitir uma visualização rápida da disposição dos elementos. O objetivo foi validar o fluxo de navegação antes de qualquer implementação técnica.

2. Arquitetura de Informação
Os wireframes definiram a hierarquia visual das seções fundamentais do site:
Cabeçalho Dinâmico: Posicionamento do logotipo e menu de navegação simplificado para facilitar o acesso às seções.
Apresentação Principal: Estrutura em grade para destacar o produto principal e uma chamada de ação clara.
Vitrine de Produtos: Planejamento de cards organizados para exibir fotos, preços e botões de compra de forma padronizada.
Área de Interação: Espaço dedicado para avaliações de clientes e campo de cadastro para novidades.

4. Ideação
O uso de wireframes manuais garantiu que o design final fosse funcional e direto ao ponto, focado na facilidade de uso para o cliente final e na organização clara dos itens oferecidos pela loja.

5. Planejamento de Rotas de Navegação
Durante a estruturação dos wireframes, definimos dois fluxos principais de navegação para garantir que o usuário encontre o que precisa com o mínimo de cliques:

Rota Simples (Navegação Direta): Projetada para o fluxo principal do site (Home, Sobre, Contato). Esta rota leva o usuário a seções de conteúdo estático, onde as informações são apresentadas de forma clara e linear, ideal para quem deseja conhecer a história da loja ou localizar o estabelecimento.

Rota Dinâmica (Navegação de Produto): Planejada especificamente para a vitrine e personalização. Ao clicar em um produto, o usuário entra em uma rota que carrega informações específicas daquele item (como o copo de 300ml ou 500ml). Essa estrutura permite que a interface se adapte para exibir diferentes opções de ingredientes e preços de forma personalizada, sem a necessidade de criar uma página estática para cada variação.
